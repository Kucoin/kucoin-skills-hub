# Protocol Fees & Revenue

Track fees generated by DeFi protocols and revenue accrued to token holders or treasuries, using the DefiLlama Fees API.

**Base URL:** `https://api.llama.fi`
**Authentication:** None required.

## Quick Reference

| Endpoint | Description | Required | Optional | Authentication |
|----------|-------------|----------|----------|----------------|
| `/overview/fees` (GET) | All protocols with 24h/7d/30d fees and global total chart | None | dataType, excludeTotalDataChart, excludeTotalDataChartBreakdown | No |
| `/summary/fees/{protocol}` (GET) | Fee and revenue history for a single protocol | protocol (path) | dataType | No |

## Parameters

- **dataType**: `dailyFees` (default) or `dailyRevenue`
- **excludeTotalDataChart**: `true` to omit global daily chart array (reduces payload size)
- **excludeTotalDataChartBreakdown**: `true` to omit chain-level breakdown chart
- **protocol**: Protocol slug (e.g. `uniswap`, `aave`, `gmx`)

## Response Fields (per protocol, `/overview/fees`)

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Protocol display name |
| `slug` | string | URL-safe identifier (use in `/summary/fees/{protocol}`) |
| `total24h` | number | Fees generated in last 24h (USD) |
| `total7d` | number | Fees generated in last 7 days (USD) |
| `total30d` | number | Fees generated in last 30 days (USD) |
| `category` | string | Protocol category (e.g. `DEX`, `Lending`) |
| `chains` | string[] | Chains the protocol operates on |

## Notes

- **Fees** = total paid by users. **Revenue** = subset retained by the protocol/treasury.
- Use `dataType=dailyRevenue` to switch from fees to revenue metrics.
- Use `excludeTotalDataChart=true` to speed up response for large overviews.
